# Matthews Terminal — Multi-Agent Voice Orchestrator

## The Vision

A voice-controlled command centre that lets you orchestrate multiple AI coding agents across different projects simultaneously, all from your phone.

You speak. Agents work. You hear what's happening.

## What Makes This Different

Every multi-agent coding tool on the market (Superset, Claude Code Agent Teams, Claude Squad, Cursor, JetBrains Air, Codex, GitHub Copilot) is keyboard-and-screen only. None of them have:

- Voice input with audio talk-back
- Phone remote control (work from your couch)
- Real-time tool status streaming to your phone
- Multi-agent orchestration controlled by voice

Matthews Terminal already has the first three working for a single session. The missing piece is multi-agent orchestration.

- **Cursor** — Basic voice-to-text. 8 parallel agents but no voice orchestration.

- **JetBrains Air** — No voice.
- **Superset** — No voice.
- **Claude Squad** — No voice.
- **Claude MPM** — No voice.

**Nobody has voice-controlled multi-agent orchestration across different projects. This is an open market.**

## How It Would Work

1. You open the Matthews Terminal app on your phone
2. You say: "Start an agent on the markets dashboard and another one on Matthews Terminal"
3. The system spawns two Claude Code CLI processes on your laptop, each in their own project folder
4. Your phone shows an agent roster: Agent 1 (Markets) — running, Agent 2 (Terminal) — running
5. You say: "Agent 1, fix the modal styling on mobile"
6. Agent 1 starts working. You hear: "Agent 1 is reading the index file... now editing the modal CSS..."
7. You say: "What's Agent 2 doing?"
8. You hear: "Agent 2 is idle, waiting for a command"
9. Agent 1 finishes: "Agent 1 is done. Fixed the modal padding and pushed to GitHub"
10. You say: "Agent 2, run the tests"
11. Both agents work independently. You get updates from both in one feed.
